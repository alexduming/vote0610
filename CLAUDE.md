# 课堂投票站（vote0610）

## 项目用途
课堂练习作品的全体投票站，最终选出 TOP3 最佳作品。
- 老师讲台导入作品名单 → 全班在手机/电脑上投票 → 大屏/老师手机实时看 TOP3 排名。

## 技术栈
- **单文件 HTML**：HTML + CSS + JavaScript 全部内联在 `index.html` 一个文件里
- **无框架、无构建工具**：双击 `index.html` 即可使用
- **数据存储**：浏览器 `localStorage`
- **目标浏览器**：Chrome / Edge / Safari（iOS 14+ / Android 8+）

## 命名规范
- **JS 变量 / 函数**：驼峰（`loadWorks`, `toggleVote`, `myVotes`）
- **CSS 类名**：kebab-case（`.vote-card`, `.ranking-item`）
- **localStorage 键**：小写下划线 + 项目前缀（`vote0610_works`, `vote0610_myVotes`）
- **DOM ID**：小写连字符（`#input-textarea`, `#vote-grid`）

## 注释要求
- 大美女无编程背景，注释必须**充分、口语化**：
  - 每个函数上方写「这个函数干什么用、为什么这么写」
  - 关键判断分支写「这里在防什么 / 处理什么场景」
  - 复杂逻辑拆成多步，每步写一句中文说明
- 历史记录总结用中文
- 代码内的技术术语保持英文（API、localStorage 等）

## 关键设计原则
1. **不依赖网络**：所有逻辑在本地跑，断网也能用
2. **移动端优先**：投票人主要用手机，触控区 ≥ 44px
3. **数据稳定性**：作品 ID 基于名称哈希生成，重新导入同名作品不会丢失已投状态
4. **渐进增强**：核心功能（投/取消）不依赖 JS 框架外的任何东西

## 改动前必做
1. 解释「为什么这么改」
2. 说明影响范围（哪些文件、哪些函数）
3. 改完用 `node --check` 验证 JS 语法（PowerShell / Bash 均可）
